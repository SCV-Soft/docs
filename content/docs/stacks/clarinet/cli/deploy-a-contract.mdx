The first step to deploying your contracts is to first generate a deployment plan.

 ```bash title="Terminal"
clarinet deployment generate --testnet
```

<Callout title="What are deployment plans?" type="info">
These are `yaml` files for creating reproducible deployment steps that publish a collection of on-chain transactions and contracts to a <TooltipProvider inline><Tooltip><TooltipTrigger asChild><span className='cursor-default border-b border-dotted border-primary'>network</span></TooltipTrigger><TooltipContent>Available networks are `devnet`, `testnet`, and `mainnet`</TooltipContent></Tooltip></TooltipProvider>.
</Callout>

Once you've reviewed your deployment plan at `deployments/<default.<network>-plan.yaml`, you can use the `apply` command to handle the deployments of your contracts.

```bash title="Terminal"
clarinet deployment apply --testnet 
```

<Callout title="Dive deeper" type="info">
Check out the [Create deployment plans](../guides/create-deployment-plans) guide to learn more on how to customize your deployment plans.
</Callout>
